# Security Policy

## Reporting a Vulnerability

We take the security of Gallium seriously. If you believe you have found a
security vulnerability, please report it as described below.

**Do not report security vulnerabilities through public GitHub issues.**
Instead, use GitHub's private vulnerability reporting feature at the
Security tab of this repository.

You should receive a response within 48 hours. If you do not receive a
response within that time, please follow up through the same channel to
confirm your original report was received.

## Supported Versions

| Version | Supported |
| ------- | --------- |
| 0.1.x   | Yes       |

## Disclosure Process

When a security bug is reported, we will:

1. Confirm the problem and determine the affected versions.
2. Audit code to find any potential similar problems.
3. Prepare fixes and release them as soon as possible.
